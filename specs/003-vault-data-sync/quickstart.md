# 快速验证: Vault 数据同步

## 准备

- 确认本地 Supabase 环境可用，已包含 Vault 数据表结构。
- 设置必要的环境变量（如本地 Supabase URL 与密钥）。
- 确认代理 `127.0.0.1:9090` 可用（如需访问外部数据源）。

**建议环境变量**:

- `SUPABASE_URL`: 本地 Supabase 实例地址
- `SUPABASE_SECRET_KEY`: 本地 Supabase 服务密钥
- `HTTP_PROXY` / `HTTPS_PROXY`: `http://127.0.0.1:9090`

## 核心验证场景

1. **首次同步**
   - 触发一次同步任务（调用 `/vaults/sync`）。
   - 验证 Vault 基础信息、成交记录、仓位与存款人数据均写入本地数据库。
   - 检查同步记录（来源、时间、成功数量）。

2. **重复同步**
   - 在无新数据的情况下再次触发同步。
   - 验证重复 Vault 地址被覆盖更新且无重复记录。

3. **回退数据源**
   - 模拟官方数据源不可用。
   - 验证回退数据源仍能写入关键字段并标注来源。

4. **部分失败**
   - 模拟部分 Vault 数据缺失或接口超时。
   - 验证同步记录中包含失败列表与原因摘要。

5. **WebSocket 实时更新**
   - 对年化收益 Top10 Vault 启用 WebSocket 监听。
   - 验证最新成交被实时写入并可查询。
