### **一、通用指南**

_(General Guidelines)_  
需 AI 遵守的通用规则，例如：

- **布局原则/Layout**：仅在必要时使用绝对定位（absolute positioning），默认优先采用响应式布局（responsive layout），并基于 Flexbox 和 Grid 构建结构化布局。
- **代码规范/Code**：实时重构代码以保持简洁性（Refactor as you go）。
- **文件管理/File Structure**：控制文件体积，将辅助函数和组件拆分为独立文件（Keep file sizes small）。

### **二、设计系统指南**

#### **2.1 按钮组件规范**

##### **2.1.4 交互动效规则**

_（Button Interaction Effects）_

| **状态**          | **视觉规范**                                                                           | **参数要求**                                   |
| ----------------- | -------------------------------------------------------------------------------------- | ---------------------------------------------- |
| **默认状态**      | ✅ 背景色/边框色需呈现平滑渐变过渡效果                                                 | 渐变方向：水平（90°）<br>颜色过渡时间：0.3s    |
| **悬停 (Hover)**  | ✅ 元素整体放大至原尺寸 1.02 倍<br>✅ 背景色加深 10%（亮度降低）                       | 缩放时间：0.3s<br>亮度变化时间：0.3s           |
| **点击 (Active)** | ✅ 元素整体缩小至原尺寸 0.98 倍<br>✅ 增加内凹阴影效果（模拟按压感）                   | 缩放时间：0.1s<br>阴影过渡时间：0.2s           |
| **动效全局约束**  | ✅ 所有状态变化必须使用流畅过渡动画<br>✅ 动画曲线：`cubic-bezier(0.25, 0.1, 0.25, 1)` | 动画总时长：≤0.3s<br>禁用弹性动画（overshoot） |

---

##### **2.1.5 多语言布局适配规则**

_（Multilingual Layout）_

| **语言环境**     | **布局要求**                                                                                       |
| ---------------- | -------------------------------------------------------------------------------------------------- |
| **中文 (zh-CN)** | 按钮最小宽度：100px<br>字间距：-0.3px（紧凑排版）<br>优先使用中文字体栈（如 Noto Sans SC）         |
| **英文 (en-US)** | 按钮最小宽度：120px（预留文本扩展空间）<br>字间距：正常<br>优先使用西文字体栈（如 Helvetica Neue） |
| **通用规则**     | 禁止固定宽度（必须使用 min-width + padding）<br>高度统一：40px（含上下 padding）                   |

---

##### **2.1.6 动效性能与兼容性**

_（Performance & Compatibility）_

1. **硬件加速要求**
   - 缩放动效必须使用 `transform` 属性实现
   - 禁用通过 `width/height` 属性触发布局重绘

2. **渐变渲染限制**
   - 渐变背景仅允许使用 CSS 线性渐变（禁止图片渐变）
   - 渐变颜色数 ≤ 2（避免复杂渐变影响性能）

3. **禁用状态规范**
   - 禁用按钮需移除所有动效（包括 hover/active）
   - 视觉透明度：60%

### **三、web3相关规则**

_(Web3 rules)_  
遵守的web3相关规则，例如：

- **web3地址展示**：如果碰到web3的地址，那么仅采用0x1234...7890这样的格式，0x后4位数字，最后4位数字。
- **web3地址链接**：只有有web3的地址，点击后均可以复制。
